# Accessibility Audit

Every grab carries a11y context for the target element, and the floating button ships a page-wide scanner you can run on demand.

## Per-grab: `A11yInfo`

When an element is grabbed, its result includes:

```ts
interface A11yInfo {
  attributes: A11yAttribute[];
  audit: A11yAuditItem[];
  hasA11y: boolean;
}

interface A11yAuditItem {
  severity: "warning" | "info";
  message: string;
}
```

- `hasA11yAttributes()` is the fast path used on mousemove. It only asks whether the element has any a11y-relevant attribute.
- `extractA11yInfo()` runs the full extraction and audit on grab.

## Page-wide scan: `scanPageA11y()`

The floating button's A11y panel calls `scanPageA11y()` to walk Vue-owned elements and run built-in checks:

| Rule                     | Flags                                                                |
| ------------------------ | -------------------------------------------------------------------- |
| Missing label            | Form controls without `<label>`, `aria-label`, or `aria-labelledby`  |
| Missing alt text         | `<img>` without `alt`                                                |
| Missing interactive name | `<button>` or `<a>` with no text, `aria-label`, or `aria-labelledby` |
| Keyboard access hint     | Custom interactive roles that may need `tabindex`                    |
| Hidden focusable element | `aria-hidden="true"` on focusable elements                           |

The panel groups findings by Vue component so you can jump to the component that owns the offending elements.

## Running the scan

Open the floating button and switch to the A11y panel. Clicking "Scan" walks the Vue component tree and renders a grouped report.

## Tips for agent hand-off

Paste `result.a11y` into your chat prompt so agents can read ARIA attributes directly and propose fixes. For whole-page audits, pipe `scanPageA11y()` to JSON and paste the grouped summary.
